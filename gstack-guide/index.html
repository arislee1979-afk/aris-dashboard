01 · 原理
01 · Concept

什麼是 gstack？

What is gstack?

Y Combinator CEO Garry Tan 開源的 Claude Code skill pack。核心概念只有一個：不要讓 AI 用同一個腦袋做所有事。

An open-source Claude Code skill pack by Y Combinator CEO Garry Tan. One core idea: don't let AI use the same mental mode for everything.

問題在哪
The Problem

沒有 gstack 的 Claude Code

你在 Claude Code 裡打：「幫我 review 這段 code」，它有時檢查安全性，有時檢查可讀性，有時隨便說幾句。每次品質不穩定，因為它不知道現在的角色是什麼。你打「幫我規劃這個功能」，它可能直接開始寫 code，跳過所有產品層面的問題。

核心問題

Claude Code 被當成萬能泥巴，捏成什麼形狀取決於你的心情。沒有結構，沒有角色，沒有一致性。

Claude Code Without gstack

You type "review my code" in Claude Code. Sometimes it checks security, sometimes readability, sometimes it barely says anything. Quality is inconsistent because it has no defined role. You type "plan this feature" and it starts writing code immediately, skipping all product-level thinking.

The Core Problem

Claude Code is treated as an all-purpose tool shaped by your mood. No structure, no role, no consistency.

解法
The Solution

gstack 給 Claude Code 明確的角色

你在 Claude Code 裡打 /review，它立刻切換成 Senior Code Reviewer 模式：「我的工作只有一件事，找 production bugs 和安全漏洞。我不管 UI，不管文件，不管效能。」每次同樣的標準，不是今天嚴格明天鬆散。

gstack Gives Claude Code a Defined Role

You type /review in Claude Code and it instantly switches to Senior Code Reviewer mode: "My only job is finding production bugs and security vulnerabilities. Not UI, not docs, not performance." Same standard every time — not strict today, loose tomorrow.

核心哲學
Core Philosophy
Garry Tan 的原話
Garry Tan's Words

「Planning 不是 Review。Review 不是 Shipping。把這三件事混在一起，你會得到一個平庸的結果。gstack 給你明確的齒輪切換——你可以告訴 Claude Code『現在我需要哪種大腦』。」

"Planning is not Review. Review is not Shipping. Blur them together and you get mediocre output. gstack gives you explicit gear shifts — you tell Claude Code which brain you need right now."

安裝教學

Installation Guide

兩個層級的概念，先懂再動手。需求：Claude Code + Git + Bun v1.0+。

Two-level concept — understand first, then install. Requirements: Claude Code + Git + Bun v1.0+.

概念先懂
The Two Levels
全域層（只裝一次）
Global Layer (once)

~/.claude/skills/gstack

gstack 原始碼。整台電腦只存一份，等於買了一套工具放在車上。

gstack source code. One copy per machine — like keeping your tools in the car.

專案層（每個專案一次）
Project Layer (per project)

your-project/.claude/skills/gstack

從全域 copy 一份進專案。每個要用的專案各做一次，等於把工具帶進工地。

Copied from global into each project. Like carrying your tools into the worksite.

第一步：全域安裝（只做一次）
Step 1: Global Install (once only)

在 Terminal（不是 Claude Code）貼這個：

In Terminal (not inside Claude Code), run:

git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup

看到 gstack ready (claude). 就成功。它會自動下載 Playwright 瀏覽器（/qa 和 /browse 需要）。

You'll see gstack ready (claude). when done. It auto-downloads a Playwright browser for /qa and /browse.

第二步：加入專案（每個專案一次）
Step 2: Add to a Project (per project)
cd ~/your-project && cp -Rf ~/.claude/skills/gstack .claude/skills/gstack && rm -rf .claude/skills/gstack/.git && cd .claude/skills/gstack && ./setup && echo "✅ Done"
1
cp：複製 gstack 進專案
cp: Copy gstack into project
cp -Rf 把全域的 gstack 整個複製到專案的 .claude/skills/。
cp -Rf copies the global gstack into your project's .claude/skills/.
2
rm -rf .git：移除 git 歷史
rm -rf .git: Remove git history
gstack 帶著自己的 git 歷史，會干擾你的 repo，所以移除。
gstack carries its own git history which would conflict with your repo — remove it.
3
./setup：在專案裡初始化
./setup: Initialize in project
讓 Claude Code 認識這個專案裡的 gstack skill 設定。
Registers gstack skills so Claude Code recognizes them in this project.
第三步：進入 Claude Code 確認
Step 3: Verify in Claude Code
cd ~/your-project
claude
Claude Code 操作
Claude Code Action

進入 Claude Code 後，打 / — 看到 /qa、/review、/ship 出現在選單就完成了。

Once inside Claude Code, type / — if you see /qa, /review, /ship in the menu, you're done.

看不到指令？
Commands not showing?

舊 session 不認識新 skill。用 /exit 或 Ctrl+C 退出，重新開 claude。還是沒有：進到 .claude/skills/gstack 重跑 ./setup。

Old sessions don't recognize new skills. Exit with /exit or Ctrl+C and reopen claude. Still missing: go to .claude/skills/gstack and rerun ./setup.
